# CopilotFleet — Полное руководство пользователя

CopilotFleet — визуальная платформа оркестрации AI-агентов. Вы строите графы из нод (узлов),
соединяете их связями и запускаете выполнение. Каждая нода представляет агента, модель,
логический блок или инструмент.

## 1. Установка и запуск

### Предварительные требования

| Компонент | Версия |
|-----------|--------|
| Node.js   | 20+    |
| pnpm      | 9+     |

### Шаги установки

```bash
# Клонируйте репозиторий
git clone <url-репозитория>
cd copilot-fleet

# Установите зависимости
pnpm install

# Соберите все пакеты
pnpm build

# Запустите веб-интерфейс в dev-режиме
pnpm --filter @copilot-fleet/web dev
```

После запуска откройте в браузере `http://localhost:5173`.

## 2. Обзор интерфейса

Интерфейс состоит из следующих областей:

```
┌─────────────────────────────────────────────────────────┐
│                    ТУЛБАР (верхняя панель)               │
├──────────┬──────────────────────────────┬───────────────┤
│          │                              │               │
│ БОКОВАЯ  │       КАНВАС (холст)         │  ИНСПЕКТОР    │
│ ПАНЕЛЬ   │    основная рабочая область  │  (правая      │
│ (палитра │    с нодами и связями        │   панель)     │
│  нод)    │                              │               │
│          │                              │               │
├──────────┴──────────────────────────────┴───────────────┤
│                    КОНСОЛЬ (внизу)                       │
├─────────────────────────────────────────────────────────┤
│                    СТАТУС-БАР                            │
└─────────────────────────────────────────────────────────┘
```

### Тулбар (верхняя панель)

Расположена в самом верху. Содержит:

- **CopilotFleet** — логотип и название приложения
- **Запуск** (Run) — запускает выполнение графа
- **Стоп** (Stop) — останавливает текущую сессию
- **Сохранить** (Save) — сохраняет граф в localStorage
- **Загрузить** (Load) — загружает последний сохранённый граф
- **Шаблоны** (Templates) — открывает выпадающее меню с шаблонами графов
- **Настройки** (Settings) — открывает модальное окно настроек
- **Имя графа** — отображается справа в тулбаре
- **Таймер** — показывает время выполнения текущей сессии
- **Прогресс** — сколько нод завершено из общего числа

## 3. Боковая панель — палитра нод

Левая панель содержит все доступные ноды, сгруппированные по секциям.

### Секции

| Секция       | Описание |
|--------------|----------|
| **Триггеры** | Ноды, которые запускают граф (Ручной Триггер) |
| **Агенты**   | Встроенные AI-агенты (Кодер, Ревьюер, Тестировщик и др.) |
| **Мои агенты** | Пользовательские агенты, созданные вами |
| **Модели**   | Прямой вызов LLM-модели без логики агента |
| **Логика**   | Условие, Разветвитель, Слияние, Группа |
| **Инструменты** | Инструменты и блок Human-in-the-loop |
| **Вывод**    | Нода вывода результата |
| **Недавние сессии** | История сохранённых графов |

### Как добавить ноду на канвас

**Способ 1 — Клик:** Нажмите на элемент в палитре — нода появится на канвасе.

**Способ 2 — Drag & Drop:** Перетащите элемент из палитры на канвас в нужную позицию.

### Поиск

В верхней части палитры есть поле поиска. Введите текст — палитра автоматически
отфильтрует ноды и агентов по названию.

### Сворачивание панели

Нажмите кнопку `<` в правом верхнем углу палитры, чтобы свернуть панель.
Повторное нажатие развернёт её обратно.

## 4. Работа с канвасом

Канвас — основная рабочая область, где вы строите граф.

### Навигация

| Действие | Управление |
|----------|------------|
| Перемещение по канвасу | Зажмите ЛКМ на пустой области и тяните |
| Масштабирование | Колёсико мыши / Ctrl+колёсико |
| Выделение ноды | Клик по ноде |
| Множественное выделение | Зажмите Shift + клик по нодам |
| Удаление | Выделите ноду, нажмите Delete / Backspace |

### Соединение нод

1. Наведите курсор на **порт** (маленький кружок на левом или правом краю ноды)
2. Зажмите ЛКМ на порту-источнике (правый край)
3. Перетащите линию к порту-приёмнику (левый край) другой ноды
4. Отпустите — связь создана

Связи показывают поток данных между нодами. Анимированные связи означают,
что данные активно передаются (нода в статусе «выполняется»).

### Мини-карта

В правом нижнем углу канваса — мини-карта для быстрой навигации по большим графам.
Можно включить/выключить в Настройки → Общие → Мини-карта.

### Сетка и привязка

- **Привязка к сетке** (Snap to Grid) — ноды автоматически выравниваются
- **Показать сетку** — отображает точечную сетку на фоне
- **Размер сетки** — настраивается в шаге от 8 до 64 пикселей

## 5. Типы нод

### Триггер (Trigger)

Стартовая нода графа. Запускает выполнение цепочки.

- **Режим**: ручной (по нажатию кнопки Run) или по расписанию
- **Иконка**: ▶ (Play)
- **Порты**: один выход (trigger-out)

### Агент (Agent)

AI-агент, выполняющий задачу с помощью LLM-провайдера.

- **Параметры**: провайдер, модель, системный промпт, температура, макс. токенов
- **Иконка**: зависит от типа агента (каждый встроенный агент имеет свою иконку)
- **Порты**: вход (agent-in) и выход (agent-out)

### LLM (Языковая модель)

Прямой вызов LLM без логики агента. Удобно для одноразовых запросов.

- **Параметры**: провайдер, модель, промпт, температура
- **Иконка**: мозг (Brain)
- **Порты**: вход и выход

### Условие (Condition)

Ветвление по условию. Оценивает выражение и направляет поток в ветку «True» или «False».

- **Параметры**: выражение условия
- **Иконка**: ветвление (GitBranch)
- **Порты**: вход, True-выход, False-выход

### Разветвитель (Splitter)

Разделяет поток на несколько параллельных путей (от 2 до 5 ветвей).

- **Параметры**: количество ветвей, имена ветвей
- **Иконка**: разделение (Split)
- **Порты**: вход и несколько выходов (A, B, C, D, E)

### Слияние (Merger)

Ожидает результаты от нескольких входных потоков и объединяет их.

- **Параметры**: стратегия слияния (all / any / race)
- **Иконка**: слияние (Merge)
- **Порты**: два входа (left, right), один выход

### Human-in-the-loop

Блок, требующий ручного одобрения человеком перед продолжением.

- **Параметры**: инструкция, таймаут
- **Иконка**: человек (User)
- **Порты**: вход и выход
- **Действия**: кнопки «Одобрить» и «Отклонить» на ноде

### Инструмент (Tool)

Вызов внешнего инструмента или скрипта.

- **Параметры**: имя инструмента, параметры
- **Иконка**: гаечный ключ (Wrench)
- **Порты**: вход и выход

### Вывод (Output)

Конечная нода графа. Собирает и отображает результат.

- **Параметры**: тип вывода (text / json / file)
- **Иконка**: файл на выходе (FileOutput)
- **Порты**: один вход

### Группа (Group)

Визуальный контейнер для группировки нод в логические блоки.

- **Иконка**: контейнер (Boxes)
- **Минимальный размер**: 400×180 пикселей

## 6. Встроенные агенты

CopilotFleet предлагает 10 предварительно настроенных агентов:

| Агент | Иконка | Провайдер | Модель | Назначение |
|-------|--------|-----------|--------|------------|
| **Кодер** | Bot | GitHub Copilot | claude-sonnet-4 | Реализация кода по плану. Пишет чистый, идиоматичный код. |
| **Ревьюер** | Shield | GitHub Copilot | claude-sonnet-4 | Ревью диффов: находит ошибки, уязвимости, регрессии. |
| **Тестировщик** | Flask | OpenAI | o3 | Пишет unit/integration тесты, анализирует покрытие. |
| **Рефакторер** | Wrench | GitHub Copilot | gpt-4.1 | Упрощает код без изменения поведения. |
| **Документатор** | FileText | OpenAI | gpt-4.1-mini | Документация API, README, ADR. |
| **Безопасность** | Lock | GitHub Copilot | o3 | OWASP-аудит, моделирование угроз, CVE-проверки. |
| **Дизайнер** | Palette | OpenAI | gpt-4.1 | UX/UI, WCAG, отзывчивый дизайн, UI-копирайтинг. |
| **DevOps** | Settings | GitHub Copilot | claude-sonnet-4 | CI/CD, контейнеры, IaC, деплой. |
| **Исследователь** | Search | OpenAI | o4-mini | Анализ кодовой базы, исследование зависимостей. |
| **Планировщик** | Map | GitHub Copilot | claude-sonnet-4 | Декомпозиция задач на фазы с критериями приёмки. |

### Системные промпты

Каждый агент имеет детальный системный промпт, определяющий его поведение, правила
и ограничения. Промпт можно посмотреть и отредактировать в Инспекторе ноды
(поле «Промпт»).

### Навыки (Skills)

Каждый агент имеет набор навыков, описывающих его компетенции:

- **Кодер**: code-generation, bug-fixing, refactoring, git-operations
- **Ревьюер**: code-review, security-audit, regression-analysis, style-check
- **Тестировщик**: unit-testing, integration-testing, coverage-analysis, tdd
- **Рефакторер**: refactoring, dead-code-removal, pattern-extraction, complexity-reduction
- **Документатор**: api-docs, readme-generation, adr-writing, comment-quality
- **Безопасность**: vulnerability-scan, threat-modeling, dependency-audit, owasp-check
- **Дизайнер**: ui-design, accessibility, responsive-layout, ux-writing
- **DevOps**: ci-cd, containerization, monitoring, infrastructure-as-code
- **Исследователь**: codebase-analysis, dependency-research, architecture-review, evidence-gathering
- **Планировщик**: task-decomposition, dependency-analysis, estimation, risk-assessment

## 7. Создание пользовательского агента

1. В боковой панели раскройте секцию **«Мои агенты»**
2. Откройте **Редактор агента** (доступен через Инспектор при выделенной ноде Agent)
3. Заполните поля:

| Поле | Описание | Пример |
|------|----------|--------|
| **Имя (name)** | Уникальный идентификатор | `security-reviewer` |
| **Отображаемое имя** | Человекочитаемое название | `Security Reviewer` |
| **Описание** | Краткое описание роли | `Audits changes for security.` |
| **Иконка** | Идентификатор иконки | `bot`, `shield`, `lock`, и др. |
| **Цвет** | HEX-цвет для карточки | `#ef4444` |
| **Провайдер** | Выберите из списка | GitHub Copilot, OpenAI, Anthropic |
| **Модель** | Модель LLM | `claude-sonnet-4`, `gpt-4.1` |
| **Системный промпт** | Детальная инструкция для агента | Описание роли, правила поведения |
| **Температура** | Креативность (0 = точный, 1 = креативный) | `0.3` |
| **Макс. токенов** | Лимит длины ответа | `4096` |
| **Таймаут** | Максимальное время работы | `30 мин` |
| **Include-паттерны** | Файлы для анализа | `src/**/*.ts` |
| **Exclude-паттерны** | Исключения | `dist/**` |
| **Before-команда** | Запуск перед работой агента | `pnpm lint` |
| **After-команда** | Запуск после работы агента | `pnpm test` |
| **Метки** | Теги для фильтрации | `security`, `review` |

4. Нажмите **«Сохранить агента»** — он появится в секции «Мои агенты»

### Предпросмотр ноды

Справа от формы отображается предпросмотр карточки агента — как она будет выглядеть
на канвасе. Обновляется в реальном времени по мере заполнения формы.

## 8. Шаблоны графов

Шаблоны — готовые конфигурации графов для типичных сценариев.

### Как использовать

1. Нажмите кнопку **«Шаблоны»** в тулбаре
2. Выберите шаблон из выпадающего меню:
   - **Quick Fix** — один агент для быстрого исправления
   - **Feature Squad** — пайплайн: планирование → код → ревью
   - **Fullstack Team** — команда из 6 агентов (план, код, дизайн, тест, мерж, ревью)
   - **Empty** — пустой канвас

3. Или откройте **Галерею шаблонов** для расширенного выбора с описаниями и тегами

### Галерея шаблонов

Галерея отображает 5 шаблонов в виде карточек:

| Шаблон | Агентов | Описание |
|--------|---------|----------|
| Quick Fix | 1 | Один агент для быстрого патча |
| Feature Squad | 3 | Пайплайн: план → код → ревью |
| Fullstack Team | 6 | Полная команда разработки |
| Refactor Platoon | 5 | Рефакторинг с безопасностью и валидацией |
| Security Audit | 3 | Аудит с условным ремедиацией и ревью |

## 10. Провайдеры и API-ключи

Провайдеры — это подключения к LLM-сервисам, которые выполняют запросы агентов.

### Поддерживаемые провайдеры

| Провайдер | Модели по умолчанию | Требует API-ключ? |
|-----------|---------------------|-------------------|
| **GitHub Copilot** | claude-sonnet-4, gpt-4o, o3-mini | Да (через GitHub Auth) |
| **OpenAI** | gpt-4o, gpt-4.1, o3-mini | Да |
| **Anthropic** | claude-sonnet-4, claude-3-7-sonnet | Да |
| **Ollama** | llama3.2, qwen2.5-coder, mistral-small | Нет (локальный) |
| **LM Studio** | local-model, qwen2.5-coder | Нет (локальный) |
| **Custom API** | custom-model | Да |
| **VS Code Local** | copilot-local | Нет |

### Как добавить провайдера

1. Откройте **Настройки → Провайдеры**
2. Внизу списка выберите тип провайдера из выпадающего меню
3. Нажмите **«Добавить провайдера»**
4. Появится карточка нового провайдера
5. Нажмите **«Редактировать»** — откроется панель редактирования

### Настройка провайдера

В панели редактирования провайдера заполните:

- **Имя** — произвольное название (например, «OpenAI Production»)
- **API-ключ** — вставьте ключ (Ctrl+V). Ключ маскируется точками после сохранения
- **Base URL** — для Ollama/LM Studio/Custom (по умолчанию `http://localhost:11434`)
- **Модели** — список доступных моделей (можно добавлять и удалять)
- **Модель по умолчанию** — выберите из списка

### Тестирование

Нажмите кнопку **«Тест»** на карточке провайдера. Зелёный индикатор — подключено,
красный — проблема (проверьте API-ключ и URL).

### Безопасность

> API-ключи хранятся в localStorage браузера. Они скрыты в интерфейсе
> и никогда не показываются полностью после сохранения.

## 11. GitHub-авторизация

Для использования провайдера GitHub Copilot необходимо авторизоваться.

### Шаги

1. Откройте **Настройки → Провайдеры**
2. В верхней части отображается блок GitHub Auth
3. Введите **Personal Access Token (PAT)** с правами `copilot`
4. Нажмите **«Подключить»**
5. После успешного подключения отобразится ваше имя пользователя и аватар

### Как создать GitHub PAT

1. Перейдите в GitHub → Settings → Developer settings → Personal access tokens
2. Нажмите **«Generate new token (classic)»**
3. Выберите scope: `copilot`
4. Скопируйте токен и вставьте в CopilotFleet

### Отключение

Нажмите **«Отключить»** в блоке GitHub Auth. Токен будет удалён из localStorage.

## 12. Инспектор ноды

Правая панель, которая отображается при выделении ноды на канвасе.
Видна на широких экранах (≥1280px).

### Поля инспектора

| Поле | Описание |
|------|----------|
| **Метка (Label)** | Имя ноды, отображаемое на карточке |
| **Описание** | Текстовое описание роли ноды |
| **Статус** | idle / queued / running / done / error / skipped / cancelled |
| **Прогресс** | Процент выполнения (0–100%) |

### Конфигурация нод по типу

В зависимости от типа ноды, инспектор показывает дополнительные поля:

**Агент (Agent):**
- Провайдер и модель
- Системный промпт
- Температура, макс. токенов
- Include/Exclude паттерны файлов

**Условие (Condition):**
- Выражение условия

**Триггер (Trigger):**
- Тип триггера (manual / scheduled)

**Инструмент (Tool):**
- Имя инструмента, параметры

### Действия

- **Дублировать ноду** — создаёт копию с теми же настройками
- **Удалить ноду** — удаляет ноду и все её связи

## 13. Консоль

Нижняя панель — лог выполнения сессии.

### Возможности

- **Фильтрация**: All / Info / Success / Error
- **Очистка**: кнопка «Очистить» удаляет все записи
- **Изменение размера**: перетащите верхнюю границу консоли вверх/вниз

### Типы записей

| Тип | Иконка | Описание |
|-----|--------|----------|
| Info | Info (i) | Информационные сообщения (старт, сохранение) |
| Success | CheckCircle | Успешное завершение |
| Error | AlertCircle | Ошибки и вызовы abort |

Каждая запись содержит timestamp, тип события и данные в JSON-формате.

## 14. Запуск и остановка сессии

### Запуск (Run)

1. Постройте граф: добавьте ноды и соедините их связями
2. Нажмите кнопку **«Запуск»** (Run) в тулбаре
3. Триггер-ноды переходят в статус «running», остальные — «queued»
4. Прогресс отображается на каждой ноде и в статус-баре внизу

### Остановка (Stop)

Нажмите **«Стоп»** (Stop) — все выполняющиеся и ожидающие ноды переходят
в статус «cancelled».

### Статусы нод

| Статус | Значение |
|--------|----------|
| idle | Простой — ожидание запуска |
| queued | В очереди — ожидает выполнения после зависимостей |
| running | Выполняется — агент или модель работает |
| done | Завершено — успешно выполнено |
| error | Ошибка — произошла ошибка |
| skipped | Пропущено — нода пропущена по условию |
| cancelled | Отменено — отменено пользователем |

## 15. Сохранение и загрузка

### Сохранение

Нажмите **«Сохранить»** в тулбаре. Граф сериализуется в JSON и сохраняется
в localStorage браузера.

### Загрузка

Нажмите **«Загрузить»** — загружается последний сохранённый граф.

### Автосохранение

Включите в Настройки → Общие → **Автосохранение**. Граф будет автоматически
сохраняться при каждом изменении.

### История сессий

В боковой панели секция **«Недавние сессии»** показывает сохранённые графы
с их названием, датой и статусом.

### Формат графа

Граф хранится в формате `FleetGraph`:

```json
{
  "id": "uuid",
  "name": "My Graph",
  "description": "...",
  "nodes": [...],
  "edges": [...],
  "metadata": {
    "version": "0.1.0",
    "createdAt": "2026-04-13T...",
    "updatedAt": "2026-04-13T..."
  }
}
```

## 17. Устранение неполадок

### Проблема: Провайдер не подключается

**Решение:**
- Проверьте API-ключ — вставьте новый через Ctrl+V
- Для Ollama / LM Studio — убедитесь, что сервер запущен (`http://localhost:11434`)
- Нажмите «Тест» — прочитайте сообщение об ошибке

### Проблема: Ноды не соединяются

**Решение:**
- Убедитесь, что тянете от порта-источника (правый круг) к порту-приёмнику (левый круг)
- Нельзя создать петлю (нода → та же нода)
- Максимальное количество связей ограничено

### Проблема: Граф не сохраняется

**Решение:**
- Проверьте, что localStorage не заполнен (очистите через DevTools → Application → Local Storage)
- Включите Автосохранение в Настройках

### Проблема: Интерфейс на английском

**Решение:**
- Откройте Настройки → Общие → Язык → выберите **RU**
- Или нажмите кнопку **EN/RU** в статус-баре внизу

### Проблема: Светлая тема выглядит неправильно

**Решение:**
- Откройте Настройки → Общие → Тема → выберите **Dark** и затем снова **Light**
- Перезагрузите страницу (F5)
